1. 코드 리팩토링
    - 외부 동작을 바꾸지 않으면서 내부 구조를 개선하는 방법
    - 코드를 가독성 있게, 심플하게 만듦

    - 중요한 점:
        * 이해하기 쉽고 수정하기 쉽도록 만드는 것
        * 겉으로 보이는 기능을 변경하지 않는 것
        * **기능 추가 X / 코드 구조 신경 O**

    - 리팩토링은 언제?
        * 기능을 추가할 때
        * 버그를 수정해야 할 때
        * 코드 검토를 할 때

    - 무엇을 리팩토링?
        * 중복된 코드
        * 긴 메소드
        * 거대한 클래스 등
